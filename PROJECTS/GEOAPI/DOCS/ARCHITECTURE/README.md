Arquitetura do backend GEOAPI - .NET 9 com Clean Architecture, CQRS e multi-tenancy via RLS PostgreSQL, dividido em 4 layers (Domain, Application, Infrastructure, Gateway) onde Domain contém entities e value objects puros sem dependências externas, Application implementa use cases via MediatR commands/queries com FluentValidation, Infrastructure provê implementações de repositórios usando EF Core + PostgreSQL + PostGIS com RLS policies e integração com Keycloak OAuth2, e Gateway expõe API REST com controllers que recebem DTOs e delegam para MediatR handlers, além de middleware de autenticação JWT Bearer que valida tokens Keycloak e extrai claims tenant_id e roles para configurar `SET LOCAL app.tenant_id` garantindo isolamento automático por tenant via RLS sem código adicional nas queries.
