# PR Guidelines

Pull Request guidelines do CARF garantindo qualidade. Template PR (título descritivo, description explaining what/why, linked issues Closes #123, checklist tests added/docs updated/no breaking changes). Code review checklist (functionality works as expected, tests adequate coverage, no security vulnerabilities, follows code style, performance acceptable, documentation updated). CI checks obrigatórios (build success, tests passing, lint no errors, coverage >= threshold). Review process (assign 2 reviewers, address comments ou discuss, approval required before merge, squash merge keeping history clean). SLA review (normal PRs 24h, urgent 4h, blocked communicate). Etiquette (reviews construtivos não blocking desnecessariamente, authors responsive feedback).
