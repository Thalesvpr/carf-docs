# Mobile Deployment

Deployment mobile REURBCAD via app stores e OTA updates garantindo distribuição controlada. iOS App Store seguindo Apple guidelines (review process 2-5 dias, TestFlight beta distribution para QA, release notes obrigatórios), code signing certificates gerenciados via Apple Developer Portal, provisioning profiles separados (development, ad-hoc, distribution), e incremento automático build number via Fastlane. Android Play Store com review mais rápido (1-2 dias), internal testing track para QA, closed beta para early adopters, production release faseado (5% → 25% → 100% usuários monitorando crash rate), e app bundles otimizando download size. Expo EAS (Expo Application Services) gerenciando builds cloud sem necessidade Xcode/Android Studio local, eas.json configurando profiles (development preview production), credentials sincronizados cross-team, e CI/CD GitHub Actions triggering builds automaticamente. OTA updates (Over-The-Air) via Expo Updates entregando JS bundle updates instantâneos sem app store review para bugfixes críticos, estratégia de rollout controlada (canary 10% usuários primeiro), rollback instantâneo se crash rate aumentar, e fallback automático para última versão estável se update corromper. Version management seguindo semver (major.minor.patch), release notes auto-gerados de Conventional Commits, e compatibility matrix documentando minimum supported version (forçar upgrade se backend quebra compatibilidade). Code push notifications alertam usuário "Nova versão disponível" com botão Atualizar Agora, force update para security patches críticos bloqueando app até instalar, e silent updates em background se mudanças minor.
