# Mobile Deployment

Deployment mobile REURBCAD via app stores e OTA updates garantindo distribuição controlada conforme [ADR-004: React Native Mobile](../ADRs/ADR-004-react-native-mobile.md). iOS App Store seguindo Apple guidelines (review process 2-5 dias, TestFlight beta distribution para QA, release notes obrigatórios), code signing certificates gerenciados via Apple Developer Portal, provisioning profiles separados (development, ad-hoc, distribution), e incremento automático build number via Fastlane. Android Play Store com review mais rápido (1-2 dias), internal testing track para QA, closed beta para early adopters, production release faseado (5% → 25% → 100% usuários monitorando crash rate), e app bundles otimizando download size. Expo EAS (Expo Application Services) gerenciando builds cloud sem necessidade Xcode/Android Studio local, eas.json configurando profiles (development preview production), credentials sincronizados cross-team, e CI/CD [GitHub Actions](../ADRs/ADR-017-github-actions-cicd.md) triggering builds automaticamente. OTA updates (Over-The-Air) via Expo Updates entregando JS bundle updates instantâneos sem app store review para bugfixes críticos, estratégia de rollout controlada (canary 10% usuários primeiro), rollback instantâneo se crash rate aumentar, e fallback automático para última versão estável se update corromper. Version management seguindo semver (major.minor.patch), release notes auto-gerados de Conventional Commits, e compatibility matrix documentando minimum supported version (forçar upgrade se backend quebra compatibilidade). Code push notifications alertam usuário "Nova versão disponível" com botão Atualizar Agora, force update para security patches críticos bloqueando app até instalar, e silent updates em background se mudanças minor.

## Projeto Mobile: REURBCAD

Deployment mobile via EAS (Expo Application Services) é específico do projeto [REURBCAD](../../../PROJECTS/REURBCAD/DOCS/README.md) aplicativo móvel React Native para coleta de dados cadastrais em campo construído conforme [ADR-004: React Native](../ADRs/ADR-004-react-native-mobile.md) com offline-first architecture via [ADR-006: WatermelonDB](../ADRs/ADR-006-offline-first-watermelondb.md) sincronizando com [GEOAPI](../../../PROJECTS/GEOAPI/DOCS/ARCHITECTURE/01-overview.md) quando dispositivo conectado WiFi, builds gerando APK Android e IPA iOS assinados com certificates gerenciados via EAS Credentials service eliminando necessidade de gerenciar provisioning profiles manualmente, updates over-the-air (OTA) via EAS Update permitindo hotfixes JS bundle sem passar por app store review cycle economizando dias de espera, e distribuição para beta testers via TestFlight (iOS) e Google Play Internal Testing (Android) antes de release público garantindo QA adequado conforme ambientes definidos em [01-environments.md](./01-environments.md).
