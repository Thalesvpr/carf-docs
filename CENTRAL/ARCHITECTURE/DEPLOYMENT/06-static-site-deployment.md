# Static Site Deployment

Deployment WEBDOCS portal VitePress como site estático em CDN garantindo performance global e custo zero/baixo. Build process executando vitepress build gerando HTML/CSS/JS estáticos pré-renderizados no build time, tree-shaking removendo código não usado, minification reduzindo bundle size, e sitemap.xml auto-gerado para SEO. Hosting options incluindo Vercel (deploy automático push main, preview deploys em PRs, edge network global, zero config), Netlify (similar Vercel, forms/functions serverless opcionais), GitHub Pages (grátis para repos públicos, Actions workflow building/deploying), CloudFront + S3 (controle total, cache TTL customizável, invalidation manual), e Nginx self-hosted (on-premises, controle absoluto, requer manutenção manual). CDN configuration com cache agressivo (static assets CSS/JS/images cache 1 ano com content hash filenames, HTML cache 5min permitindo updates rápidos), Brotli/Gzip compression reduzindo transfer 70%, HTTP/2 multiplexing reduzindo latency, e preload critical resources (fonts, hero images). CI/CD pipeline GitHub Actions triggering build em push docs changes, validando links quebrados via lychee, gerando search index Algolia DocSearch, deploying automaticamente production, e notificando Slack se build falhar. Versioning mantém múltiplas versões docs simultaneamente (/v1.0/, /v1.1/, /latest/ redirecting para current), dropdown seletor versão UI, e deprecated versions mostram banner "Esta versão está desatualizada". Analytics via Google Analytics 4 rastreando páginas mais visitadas, search queries usuários, e tempo médio página otimizando conteúdo.
