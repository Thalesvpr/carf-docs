# ADRs

Architecture Decision Records documentando decisões arquiteturais críticas do CARF com contexto (por que decisão foi necessária), opções consideradas (alternativas avaliadas com prós/contras), decisão tomada (solução escolhida), consequências (impactos positivos e negativos), e status (proposed/accepted/deprecated/superseded). Exemplos: ADR-001 .NET 9 backend (vs Node.js Java Python Go), ADR-002 PostgreSQL + PostGIS (vs SQL Server spatial vs MongoDB), ADR-003 Keycloak autenticação (vs Auth0 vs implementação própria), ADR-004 React Native mobile (vs Flutter vs native), ADR-005 multi-tenancy RLS (Row-Level Security vs schema separation vs database separation), ADR-006 offline-first WatermelonDB (vs Realm vs SQLite puro), ADR-007 Bun runtime/bundler (vs Node.js + Vite/Webpack), ADR-008 Clean Architecture + DDD (vs layered tradicional), ADR-009 CQRS pattern (vs CRUD tradicional), ADR-010 Event-Driven Architecture (vs request/response síncrono), ADR-011 shared library @carf/tscore (vs código duplicado vs monorepo). ADRs são imutáveis - novas decisões criam novos ADRs ao invés de editar existentes.
