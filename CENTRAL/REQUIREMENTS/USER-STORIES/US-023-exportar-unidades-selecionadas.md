# US-023: Exportar Unidades Selecionadas

Como gestor, quero exportar unidades filtradas para que eu gere relatórios externos, onde o sistema permite extrair dados de unidades em formatos padrão de intercâmbio permitindo análise em ferramentas externas, integração com outros sistemas, e geração de relatórios customizados, garantindo interoperabilidade e flexibilidade no uso dos dados cadastrais. O cenário principal de uso inicia quando um gestor aplica filtros desejados na listagem de unidades para selecionar conjunto específico de registros e então solicita exportação escolhendo formato de saída entre Shapefile (para uso em GIS), GeoJSON (para aplicações web) ou Excel (para análise tabular), permitindo que o sistema processe a solicitação e disponibilize arquivo para download. Os critérios de aceitação incluem suporte a múltiplos formatos de exportação sendo Shapefile com todos os componentes (.shp, .dbf, .shx, .prj) para compatibilidade com software GIS, GeoJSON para uso em aplicações web e ferramentas de mapeamento online, e Excel/CSV para análise de atributos em planilhas, aplicação de filtros onde apenas unidades que atendem aos critérios de busca/filtro ativos são incluídas na exportação, processamento assíncrono quando exportação envolve mais de 1000 unidades onde job é processado em background e usuário recebe notificação quando completo, e email de notificação automático contendo link para download quando arquivo de exportação está pronto evitando necessidade de usuário aguardar em tela de processamento. Esta funcionalidade é implementada pelos módulos GEOAPI (endpoints POST /api/exports para iniciar exportação com parâmetros de filtro e formato, e GET /api/exports/{id}/download para baixar arquivo gerado, usando GDAL para conversões de formato) e GEOWEB (botão de exportação com seleção de formato e feedback de progresso), garantindo rastreabilidade com RF-082 (Exportação de Dados) e UC-007 (Caso de Uso de Exportação), onde exportações respeitam permissões do tenant incluindo apenas dados acessíveis ao usuário, arquivos gerados expiram após período configurável, e metadados como data de exportação e filtros aplicados são incluídos.
