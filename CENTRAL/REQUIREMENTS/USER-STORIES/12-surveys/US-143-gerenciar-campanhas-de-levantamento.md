# US-143: Gerenciar Campanhas de Levantamento

Como Gerente quero criar e listar campanhas topograficas para que possa organizar trabalho de campo por campanha agrupando medicoes pontos topograficos e equipes de coleta em contextos temporais e geograficos especificos, onde o endpoint implementado em GEOAPI expoe a rota /api/surveying/campaigns permitindo operacoes GET para listagem paginada e POST para criacao de novas campanhas, garantindo validacao de permissoes baseada em roles onde usuarios com role Manager ou superior podem criar campanhas e usuarios com role Surveyor podem listar e visualizar campanhas, incluindo validacao de dados de entrada onde nome da campanha deve ser unico dentro do tenant data de inicio deve ser anterior a data de fim e communityId deve referenciar comunidade existente, permitindo filtragem da listagem por status para filtrar por PLANNING ACTIVE COMPLETED CANCELLED, communityId para campanhas de uma comunidade especifica, dateRange para filtrar por periodo de execucao e responsibleId para campanhas sob responsabilidade de topografo especifico, onde a resposta da listagem inclui paginacao com page size totalCount e array de objetos Campaign contendo id name description status startDate endDate communityId communityName responsibleId responsibleName equipmentUsed measurementsCount pointsCount createdAt updatedAt, garantindo rastreabilidade ao requisito funcional RF-157 que especifica gerenciamento de campanhas de levantamento topografico, incluindo relacionamento com entidade Community onde cada campanha pertence a uma comunidade e relacionamento com entidades Measurement e SurveyPoint onde cada campanha agrupa multiplas medicoes e pontos coletados, permitindo que o frontend GEOWEB exiba dashboard de campanhas com cards mostrando status progresso de coleta metricas de precisao e mapa com perimetro da area de levantamento, onde criacao de campanha requer payload JSON com propriedades name description communityId startDate endDate responsibleId equipmentUsed array de equipamentos topograficos utilizados como GPS RTK estacao total trena laser e methodologyNotes descricao textual da metodologia de coleta adotada, garantindo validacao de sobreposicao temporal onde sistema alerta quando nova campanha se sobrepoe a campanha ativa existente na mesma comunidade sugerindo ajuste de datas ou vinculacao a campanha existente, incluindo geofencing automatico onde sistema sugere perimetro geografico da campanha baseado em envelope das geometrias das unidades da comunidade permitindo ajuste manual pelo gerente, onde a implementacao em GEOAPI utiliza Entity Framework Core com soft delete para campanhas canceladas preservando historico e rastreabilidade de levantamentos executados, permitindo clonagem de campanhas onde gerente pode criar nova campanha baseada em configuracoes de campanha anterior copiando equipamentos metodologia e responsavel mas gerando novo ID e resetando contadores de medicoes, garantindo workflow de aprovacao onde campanhas criadas ficam em status PLANNING ate que gerente execute acao de start transition para status ACTIVE liberando coleta de dados no campo, incluindo dashboard de monitoramento em tempo real onde gerente acompanha progresso da coleta quantidade de pontos coletados por dia por topografo metricas de precisao e alertas de qualidade como pontos com DOP Dilution of Precision excessivo, onde testes unitarios validam regras de negocio incluindo unicidade de nome validacao de datas validacao de sobreposicao temporal e calculo de metricas agregadas, garantindo testes de integracao que verificam criacao listagem atualizacao cancelamento e clonagem de campanhas em banco de dados PostgreSQL com validacao de isolamento multi-tenant.
