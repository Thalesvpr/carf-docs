# US-105: Relatório Sumário de Comunidade

Como gerente de projeto responsável por comunidade específica, quero gerar relatório resumido consolidando informações principais da comunidade para que visualizar KPIs essenciais seja possível em formato conciso adequado para apresentações executivas, relatórios periódicos para stakeholders, e documentação de progresso do projeto de regularização fundiária. A funcionalidade deve compilar relatório estruturado incluindo identificação da comunidade com nome, código, localização geográfica e data de criação do cadastro, estatísticas quantitativas de unidades incluindo total cadastrado, distribuição por status (DRAFT, PENDING, APPROVED, REJECTED), e taxa de completude de informações obrigatórias. O relatório deve apresentar métricas de titulares incluindo quantidade de pessoas beneficiárias, distribuição por gênero quando informado, faixas etárias predominantes, e indicadores socioeconômicos agregados respeitando privacidade individual. Informações territoriais devem incluir área total da comunidade em hectares, densidade habitacional calculada como unidades por hectare, perímetro delimitado, e mapa de localização com features principais destacados. O sistema deve calcular indicadores de progresso incluindo percentual de unidades com geometria validada, documentação completa anexada, processos de legitimação iniciados versus concluídos, e timeline de marcos importantes (início do cadastro, primeira aprovação, metas atingidas). O relatório deve ser gerado em formato estruturado JSON para integração com outros sistemas ou apresentado em formato visual HTML/PDF para consumo humano direto. Os critérios de aceitação incluem endpoint implementado retornando objeto JSON estruturado com todas as seções do sumário (identificação, estatísticas de unidades, métricas de titulares, informações territoriais, indicadores de progresso), validação de permissões garantindo acesso apenas a usuários vinculados à comunidade ou com roles gerenciais, tratamento adequado de erros incluindo comunidade não encontrada ou dados insuficientes para cálculo de métricas, documentação OpenAPI com schema completo do relatório e exemplos, e testes unitários e de integração validando cálculos agregados e consistência de dados. Esta User Story está relacionada ao RF-200 e é implementada através do endpoint GET /api/reports/community-summary no backend GEOAPI com queries agregadas otimizadas, e interface de visualização no frontend GEOWEB com opções de exportação, pertencendo ao Epic 10: Relatórios e Exportação. O status atual é proposed, com funcionalidade implementada mas formalização documental posterior.
