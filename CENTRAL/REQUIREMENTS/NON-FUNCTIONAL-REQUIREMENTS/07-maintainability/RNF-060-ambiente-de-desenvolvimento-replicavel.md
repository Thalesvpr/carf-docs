# RNF-060: Ambiente de Desenvolvimento Replicável

O ambiente de desenvolvimento para todos os módulos do sistema GEOMAP5 deve ser configurável e completamente funcional em menos de 15 minutos, permitindo que novos desenvolvedores ou membros da equipe possam começar a trabalhar produtivamente no código sem necessitar passar horas ou dias configurando dependências, bancos de dados e serviços auxiliares. Esta configuração rápida deve ser alcançada através da utilização de Docker Compose para orquestrar todos os serviços necessários para desenvolvimento local, incluindo o banco de dados PostgreSQL com PostGIS, servidor de cache Redis, serviço de autenticação Keycloak, storage de objetos MinIO ou similar, e quaisquer outros componentes que a aplicação necessite, garantindo que com um único comando todos os desenvolvedores tenham exatamente o mesmo ambiente independentemente do sistema operacional ou configurações específicas de suas máquinas. O ambiente deve incluir dados de seed preparados especificamente para desenvolvimento, contendo exemplos representativos de todas as entidades principais do sistema, como comunidades, unidades imobiliárias, titulares, documentos e geometrias, permitindo que os desenvolvedores testem funcionalidades imediatamente sem necessitar criar manualmente dados de teste, além de garantir que todos trabalhem com datasets consistentes que facilitam reprodução de bugs e comportamentos. A documentação de setup do ambiente deve ser mantida constantemente atualizada, incluindo pré-requisitos claramente listados (como versões mínimas de Docker e Docker Compose), instruções passo-a-passo para diferentes sistemas operacionais (Windows, macOS, Linux), comandos específicos para iniciar e parar o ambiente, orientações sobre como acessar cada serviço (URLs, portas, credenciais padrão), e troubleshooting de problemas comuns que desenvolvedores possam encontrar durante o setup inicial. A capacidade de configurar o ambiente rapidamente reduz significativamente a frustração de onboarding, permite que desenvolvedores contribuam mais rapidamente, facilita a rotação de pessoas entre projetos, e garante que o ambiente de desenvolvimento espelhe fielmente o ambiente de produção em termos de arquitetura e dependências. Este requisito é classificado como should-have, sendo essencial para produtividade da equipe, redução de problemas do tipo "funciona na minha máquina", facilitação de pair programming e mob programming onde desenvolvedores alternam entre máquinas, e demonstração de profissionalismo e maturidade da engenharia de software ao eliminar barreiras técnicas que impedem contribuições rápidas ao projeto.
