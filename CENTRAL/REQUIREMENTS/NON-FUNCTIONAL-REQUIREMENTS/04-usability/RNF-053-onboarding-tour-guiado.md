# RNF-053: Onboarding

A aplicação GEOWEB deve oferecer experiência de onboarding para novos usuários através de tour guiado interativo que apresenta principais funcionalidades e reduz curva de aprendizado inicial, especialmente importante considerando que usuários podem ser funcionários públicos com diferentes níveis de familiaridade com sistemas geoespaciais. O tour interativo nas principais funcionalidades deve ser ativado automaticamente no primeiro acesso do usuário após login inicial, detectado através de flag no perfil do usuário indicando se já completou onboarding, apresentando sequência de 5-8 passos que destacam elementos-chave da interface usando overlays semi-transparentes que escurecem resto da tela e spotlights que focam atenção no elemento sendo explicado, com textos explicativos curtos e acionáveis como "Este é o mapa onde você visualiza unidades habitacionais da comunidade. Clique e arraste para navegar" ou "Use este painel para buscar unidades por endereço, titular ou número de cadastro", permitindo navegação sequencial através de botões "Próximo" e "Anterior" e indicador de progresso mostrando em qual passo usuário está (ex: 3 de 7). Os tooltips contextuais devem complementar o tour inicial oferecendo ajuda just-in-time quando usuário interage com funcionalidades menos óbvias, aparecendo automaticamente na primeira vez que usuário acessa determinada tela ou hover sobre elemento específico, usando biblioteca como react-joyride ou intro.js que gerencia posicionamento dinâmico de tooltips evitando que saiam da viewport, com mensagens breves focadas em uma única ação como "Clique aqui para adicionar nova unidade habitacional" ou "Use este filtro para visualizar apenas unidades pendentes de validação", e marcando tooltips como visualizados para não aparecer novamente em sessões futuras. A opção de pular ou desabilitar o tour é essencial para respeitar preferências dos usuários e evitar frustração, incluindo botão proeminente "Pular Tour" em cada passo que permite encerrar imediatamente a sequência, checkbox "Não mostrar isso novamente" para usuários experientes que preferem explorar por conta própria, e configuração no perfil do usuário permitindo reativar tour manualmente através de menu de ajuda caso queiram revisitar instruções, com salvamento de preferências no backend para persistir entre dispositivos e sessões. Os diferentes tipos de usuários devem potencialmente receber tours customizados baseado em seus roles, onde administradores de sistema veem tour focado em configurações globais, gestão de usuários e relatórios analíticos, coordenadores de campo recebem tour enfatizando visualização de progresso, validação de dados coletados e exportação de relatórios, enquanto técnicos de campo veem tour focado em cadastro de unidades, upload de fotos e sincronização com aplicativo mobile. A implementação deve balancear utilidade e não-intrusividade, ativando tour automaticamente apenas no primeiro acesso mas nunca forçando usuário a completá-lo, mantendo passos curtos e focados (máximo 2-3 frases por tooltip), evitando jargão técnico em favor de linguagem simples e orientada a tarefas, e considerando pular certos passos automaticamente se usuário já realizou ação correspondente organicamente. Os critérios de aceitação incluem a implementação de tour interativo com 5-8 passos cobrindo funcionalidades principais (navegação no mapa, busca de unidades, cadastro básico, visualização de relatórios), a exibição automática do tour apenas no primeiro acesso de novos usuários com detecção via flag em perfil, a disponibilização de tooltips contextuais em pelo menos 10 funcionalidades não-óbvias que aparecem na primeira interação, a presença de botão "Pular Tour" visível em cada passo e opção no perfil para desabilitar ou reativar onboarding, e feedback de pelo menos 5 usuários beta testando que tour ajudou a entender funcionalidades básicas sem ser intrusivo. Este requisito é could-have por adicionar valor para experiência de novos usuários e potencialmente reduzir demanda por treinamento formal e suporte, mas não sendo essencial para operação do sistema, especialmente considerando que treinamento presencial ou via videoconferência pode ser mais efetivo para sistemas complexos de regularização fundiária onde contexto de processo é tão importante quanto mecânica da ferramenta, podendo ser implementado após funcionalidades core estarem maduras e estáveis quando investimento em onboarding traz maior retorno.
