# RNF-084: Integração WMS/WMTS

O sistema deve consumir serviços WMS e WMTS padrão OGC garantindo capacidade de sobrepor dados do sistema com camadas cartográficas de referência disponibilizadas por órgãos oficiais e provedores de dados geoespaciais, onde integração com padrões abertos permite aproveitamento de infraestruturas existentes de dados espaciais e enriquecimento de visualizações com contexto cartográfico. A implementação deve suportar WMS versões 1.1.1 e 1.3.0 que são especificações do Open Geospatial Consortium para Web Map Service, permitindo requisição de mapas renderizados como imagens através de requisições HTTP padronizadas, onde parâmetros como bounding box, layers, estilos, e sistema de coordenadas são especificados em query string permitindo composição flexível de visualizações. O sistema deve suportar WMTS versão 1.0.0 correspondente a Web Map Tile Service que é otimização de WMS para servir mapas pré-renderizados em grades de tiles, onde cache de tiles em múltiplas escalas permite performance superior e redução de carga em servidores, sendo padrão utilizado por serviços de base map como imagens de satélite e mapas de ruas. A solução deve implementar parsing de GetCapabilities que é operação padrão OGC para descoberta de metadados de serviços, onde requisição de capabilities retorna documento XML descrevendo layers disponíveis, sistemas de coordenadas suportados, escalas, e extensão geográfica, permitindo configuração dinâmica de integrações sem hardcoding de parâmetros. A implementação deve permitir configuração de múltiplas fontes WMS e WMTS através de interface administrativa, onde usuários podem adicionar URLs de serviços externos, selecionar layers de interesse, configurar opacidade e ordem de renderização, permitindo customização de mapas base conforme necessidades específicas de cada município ou projeto. Os critérios de aceitação incluem validação de que serviços WMS e WMTS públicos como os disponibilizados pelo IBGE e serviços estaduais de cartografia podem ser adicionados e visualizados corretamente, onde tiles ou imagens são carregadas com performance adequada e reprojeção entre sistemas de coordenadas diferentes é tratada transparentemente. A prioridade é classificada como should-have considerando que integração com serviços OGC aumenta significativamente valor do sistema ao permitir overlay com bases oficiais, mas sistema pode operar inicialmente com mapas base estáticos ou serviços comerciais como OpenStreetMap.
