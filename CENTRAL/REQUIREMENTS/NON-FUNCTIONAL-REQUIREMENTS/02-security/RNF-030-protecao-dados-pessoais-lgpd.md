# RNF-030: Proteção de Dados Pessoais (LGPD)

O sistema GEOMAP5 deve estar em conformidade plena com a Lei Geral de Proteção de Dados Pessoais (LGPD - Lei 13.709/2018) do Brasil, implementando mecanismos técnicos e processuais que garantam os direitos dos titulares de dados e as obrigações dos controladores e operadores de dados pessoais. O sistema deve implementar coleta de consentimento explícito e informado para processamento de dados pessoais, onde usuários devem ser claramente informados sobre quais dados estão sendo coletados, para quais finalidades serão utilizados, quanto tempo serão retidos e com quem podem ser compartilhados, com interface que permita aceitar ou recusar o consentimento de forma granular para diferentes finalidades quando aplicável. O direito ao esquecimento deve ser implementado através de funcionalidade que permita exclusão completa e irreversível de dados pessoais de um titular mediante solicitação validada, incluindo remoção de backups ou anonimização nos casos onde retenção seja legalmente requerida, com processo auditável que demonstre que a exclusão foi efetivamente realizada em todos os sistemas e réplicas. A portabilidade de dados deve ser garantida através de endpoint de exportação que permita ao titular obter todos os seus dados pessoais em formato estruturado e legível por máquina, preferencialmente JSON ou CSV, incluindo dados cadastrais, histórico de operações, documentos anexados e quaisquer outras informações coletadas, permitindo que o titular migre seus dados para outros sistemas ou simplesmente tenha cópia para seus registros. Logs de acesso a dados pessoais devem registrar toda visualização, modificação ou exportação de informações pessoalmente identificáveis, incluindo qual usuário ou processo acessou os dados, quando o acesso ocorreu e qual a justificativa ou contexto do acesso, permitindo auditoria de compliance e investigação de acessos não autorizados. O sistema deve implementar princípio de minimização de dados, coletando apenas informações estritamente necessárias para as finalidades declaradas, com revisão periódica de campos coletados para eliminar dados desnecessários. Anonimização ou pseudonimização devem ser aplicadas sempre que possível para processamentos que não requerem identificação direta dos titulares, especialmente em análises estatísticas e relatórios agregados. Notificações de incidentes de segurança que possam resultar em risco aos direitos dos titulares devem ser enviadas à Autoridade Nacional de Proteção de Dados (ANPD) e aos titulares afetados dentro dos prazos legais estabelecidos, com processo documentado para detecção, contenção, análise de impacto e comunicação de breaches. A implementação deve incluir designação de Data Protection Officer (DPO) ou encarregado de dados com canal de comunicação acessível aos titulares para exercício de seus direitos. Este requisito é Must-have pois conformidade com LGPD não é opcional para sistemas operando no Brasil que processam dados pessoais, com multas que podem chegar a 2% do faturamento da organização limitadas a 50 milhões de reais por infração, além de danos reputacionais severos em caso de violações.
