# RNF-018: HTTPS Obrigatório

Toda comunicação entre clientes e servidores do sistema GEOMAP5 deve utilizar o protocolo HTTPS com suporte mínimo a TLS 1.2 ou superior, garantindo que todos os dados trafegados pela rede sejam criptografados e protegidos contra interceptação, man-in-the-middle attacks e outras formas de espionagem ou manipulação durante transmissão. Este requisito aplica-se aos módulos GEOAPI e GEOWEB, onde o GEOAPI deve estar configurado para aceitar apenas conexões HTTPS e o GEOWEB deve ser servido exclusivamente via HTTPS. Os critérios de aceitação especificam que o servidor deve possuir um certificado SSL válido emitido por uma autoridade certificadora confiável, sendo que certificados autoassinados ou expirados não são aceitáveis em ambiente de produção pois causam avisos de segurança nos navegadores e podem ser facilmente contornados ou falsificados, reduzindo significativamente a segurança proporcionada pela criptografia. O sistema deve implementar redirect automático de HTTP para HTTPS, onde qualquer tentativa de acesso via HTTP (porta 80) é automaticamente redirecionada para HTTPS (porta 443) através de resposta HTTP 301 ou 308, garantindo que mesmo usuários que digitam URLs sem especificar o protocolo ou seguem links antigos sejam automaticamente direcionados para a versão segura, eliminando possibilidade de comunicação não-criptografada acidental. Adicionalmente, o header HSTS (HTTP Strict Transport Security) deve estar habilitado e configurado adequadamente, instruindo navegadores a sempre utilizarem HTTPS para acessar o domínio pelo período especificado (tipicamente 1 ano ou mais), impedindo downgrade attacks onde atacantes tentam forçar comunicação via HTTP não-criptografado, sendo que o HSTS também pode incluir subdomínios e ser pré-carregado em navegadores através de listas mantidas pelos fornecedores. Este requisito é classificado como Must-have porque a proteção de dados em trânsito é fundamental para segurança e privacidade, especialmente considerando que o sistema manipula informações pessoais sensíveis sujeitas à LGPD como CPF, endereços e dados de posse, além de credenciais de autenticação que se comprometidas permitiriam acesso não autorizado completo ao sistema. A implementação deve considerar obtenção de certificados SSL/TLS através de autoridades certificadoras reconhecidas como Let's Encrypt que oferece certificados gratuitos com renovação automatizada, ou certificados comerciais de autoridades como DigiCert ou GlobalSign para ambientes corporativos, configuração de web server (nginx, IIS) ou load balancer para forçar HTTPS e implementar redirect, desabilitação completa de protocolos inseguros como SSL 3.0, TLS 1.0 e TLS 1.1 que possuem vulnerabilidades conhecidas, configuração de cipher suites fortes priorizando forward secrecy através de algoritmos como ECDHE, habilitação de OCSP stapling para validação eficiente de certificados, e monitoramento contínuo de validade de certificados com alertas antes da expiração para prevenir interrupções de serviço.
