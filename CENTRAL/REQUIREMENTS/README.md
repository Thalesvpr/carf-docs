# REQUIREMENTS

Requisitos centrais do sistema CARF organizados em três categorias: FUNCTIONAL-REQUIREMENTS (especificando o que o sistema deve fazer de forma atômica e testável), USE-CASES (documentando fluxos cross-cutting que atravessam múltiplos projetos com atores, pré-condições, pós-condições e cenários alternativos), e USER-STORIES (compartilhadas focadas em autenticação, auditoria e multi-tenancy no formato persona/ação/objetivo com critérios Gherkin). Projetos não duplicam requisitos - utilizam arquivos _INDEX gerados automaticamente que filtram por tags e linkam de volta para este diretório CENTRAL como fonte única de verdade (SSOT). Cada requisito possui rastreabilidade bidirecional conectando RFs→UCs→USs→código e metadados YAML com epic, módulo, prioridade e story points.
