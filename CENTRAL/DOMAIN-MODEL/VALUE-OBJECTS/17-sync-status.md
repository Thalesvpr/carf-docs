# SyncStatus (Status de Sincronização)

Value object enum representando resultado de tentativa de sincronização offline permitindo rastreamento e exibição apropriada de estado em interfaces mobile. Valores possíveis incluem PENDING sincronização iniciada mas ainda em andamento aguardando conclusão, SUCCESS sincronização completada com sucesso sem erros ou conflitos transferindo todos os dados planejados, CONFLICT sincronização concluída mas com conflitos detectados requerendo resolução manual antes que dados possam ser consolidados (versão local e versão servidor divergiram necessitando escolha de qual manter ou merge), e FAILED sincronização falhou completamente devido a erro técnico como falta de conexão timeout de rede erro de servidor ou dados inválidos impedindo qualquer transferência. Regras de transição estabelecem que status inicial de SyncLog é sempre PENDING quando tentativa é iniciada, pode transitar para SUCCESS se operação completa sem problemas (todos registros transferidos sem conflitos ou erros), transita para CONFLICT se operação completa mas detecta divergências entre versões local e servidor que impedem merge automático (exemplo: usuário A editou Unit offline, usuário B editou mesma Unit online, ao sincronizar A detecta conflito), transita para FAILED se erro técnico impede conclusão (rede cai durante download, servidor retorna erro 500, validação de dados rejeita payload), e não há transições posteriores (status é final imutável após ser definido, nova tentativa de sincronização cria novo SyncLog). Exibição em interfaces mobile varia por status onde PENDING mostra spinner de loading com mensagem "Sincronizando..." e possivelmente barra de progresso se volume de dados é conhecido, SUCCESS exibe notificação de sucesso (ícone verde checkmark) com mensagem "Sincronização concluída" que desaparece automaticamente após alguns segundos, CONFLICT mostra alerta laranja/amarelo com mensagem "Conflitos detectados, toque para resolver" direcionando usuário para tela de resolução manual onde pode comparar versões e escolher qual manter, e FAILED exibe erro vermelho com mensagem descritiva "Falha na sincronização: sem conexão" ou "Erro do servidor, tente novamente" incluindo botão "Tentar Novamente". Ações permitidas por status incluem PENDING não permite nova sincronização até que atual complete (botão "Sincronizar" desabilitado), SUCCESS permite iniciar nova sincronização imediatamente se usuário desejar atualizar dados novamente, CONFLICT bloqueia novas sincronizações até que conflitos sejam resolvidos (sistema pode automaticamente tentar sincronizar novamente após usuário resolver conflitos marcando registros como merged), e FAILED permite retry imediato através de botão "Tentar Novamente" que cria novo SyncLog e reinicia processo. Persistência local no mobile armazena último SyncStatus conhecido permitindo exibir estado mesmo quando app é fechado e reaberto (usuário vê que última sincronização foi CONFLICT mesmo após reiniciar app podendo resolver conflitos pendentes), badge de notificação pode mostrar número de conflitos não resolvidos chamando atenção para ação necessária, e histórico completo de sincronizações com status preserva timeline de sucessos e falhas útil para troubleshooting. Métricas agregadas permitem calcular taxa de sucesso de sincronização por usuário ou dispositivo (90% SUCCESS indica usuário com boa conectividade, 50% FAILED indica problemas de rede ou configuração), identificar padrões de conflitos (conflitos frequentes em Unit específica pode indicar múltiplos usuários editando simultaneamente requerendo melhor coordenação de equipe), e alertar sobre falhas recorrentes (usuário sem sucesso há 7 dias pode ter permissões revogadas ou app desatualizado necessitando intervenção de suporte).

**Módulos:** GEOAPI, GEOWEB, REURBCAD, GEOGIS
