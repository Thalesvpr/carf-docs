# BaseValueObject (Value Object Base)

Classe base abstrata conceitual que todos value objects herdam fornecendo implementação padrão de igualdade por valor imutabilidade e comportamentos comuns diferenciando de entities que são comparadas por identidade. Conceito fundamental estabelece que value objects não têm identificador próprio sendo definidos completamente por seus valores de campos, dois value objects com mesmos valores são considerados iguais mesmo sendo instâncias diferentes em memória, são imutáveis onde após criação campos não podem ser alterados garantindo segurança em compartilhamento e uso como chave de dicionário, e validação ocorre no construtor lançando exception se dados inválidos impedindo existência de value object em estado inválido. Comportamentos comuns incluem igualdade por valor sobrescrevendo Equals comparando todos campos componentes ao invés de referência, GetHashCode sobrescrito retornando hash baseado em valores permitindo uso correto em coleções hash, operadores == e != delegando para Equals fornecendo sintaxe natural, método abstrato GetEqualityComponents retornando campos que participam da igualdade implementado por subclasses, e ToString formatando valor de forma legível para logging e debugging. Regras de design estabelecem que campos devem ser readonly ou propriedades get-only garantindo imutabilidade após construção, validação completa no construtor com exceptions descritivas se dados inválidos tipo CPF lançando ValidationException se dígitos verificadores errados, métodos de transformação retornam NOVO value object ao invés de modificar existente tipo Email.Normalize retornando novo Email lowercase, value objects podem conter lógica de negócio relacionada ao valor tipo Cpf.Format ou GeoPolygon.CalculateArea, e comparação deve ser null-safe tratando corretamente casos onde um ou ambos operandos são null. Exemplos comuns incluem Cpf validando 11 dígitos e verificadores, Email validando formato RFC, Address agrupando logradouro número cidade criando objeto coeso, GeoPolygon validando geometria fechada sem auto-interseções, Money com valor e moeda garantindo operações aritméticas type-safe, DateRange com início e fim validando que início < fim, e enums tipados como UnitStatus CommunityType fornecendo type safety ao invés de strings mágicas. Implementação varia onde .NET usa record types com init-only properties facilitando imutabilidade, TypeScript usa readonly properties e Object.freeze, Python usa dataclass frozen=True ou NamedTuple, e validação estrutural conforme plataforma. Value objects promovem ubiquitous language encapsulando conceitos de domínio tipo usar Cpf ao invés de string torna código mais expressivo e type-safe, reduzem primitive obsession evitando passar strings números brutos que não carregam semântica, facilitam testes unitários testando lógica de validação e transformação isoladamente, e melhoram manutenibilidade centralizando regras de um conceito em único lugar.
