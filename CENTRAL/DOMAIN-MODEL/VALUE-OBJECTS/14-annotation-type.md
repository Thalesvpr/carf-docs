# AnnotationType (Tipo de Anotação)

Value object enum representando classificação de anotação definindo severidade e tratamento no fluxo de trabalho. Valores possíveis incluem NOTE anotação informativa sem impacto em workflow usado para comunicação entre usuários ou registro de informações contextuais, WARNING alerta que requer atenção mas não bloqueia progresso indicando situação que pode se tornar problema se não tratada, ISSUE problema identificado que impede progressão de workflow até ser resolvido bloqueando aprovações ou transições de status, e REMINDER lembrete de ação futura com data/hora agendada disparando notificações automáticas. Regras de tratamento estabelecem que NOTE não afeta validações ou transições de status sendo puramente informativo e opcional, WARNING é exibido com destaque visual (ícone de alerta amarelo ou laranja) em interfaces alertando usuários mas permitindo prosseguir, ISSUE com priority HIGH ou URGENT pode bloquear transição de Unit de PENDING_ANALYSIS para APPROVED ou impedir submissão de LegitimationRequest até resolução (validação verifica issues não resolvidos lançando erro com lista de issues pendentes), e REMINDER configura data/hora futura disparando notificação via email ou in-app quando timestamp é atingido lembrando usuário de revisitar entidade ou executar ação. Exibição em interfaces varia por tipo onde NOTE é mostrado em timeline cronológica com ícone de comentário cinza, WARNING aparece em seção destacada de alertas com ícone de exclamação amarelo e texto explicativo, ISSUE é listado em seção de bloqueadores com ícone de erro vermelho e botão "Resolver" habilitado apenas para usuários com permissão, e REMINDER aparece em calendário ou lista de tarefas agendadas com data de vencimento visível. Criação de anotações permite que qualquer usuário com acesso de leitura à entidade crie NOTE documentando observações, usuários com acesso de edição podem criar WARNING sinalizando inconsistências detectadas, apenas ANALYST ou superior podem criar ISSUE oficialmente bloqueando progresso (field agents podem criar NOTE pedindo revisão mas não ISSUE bloqueante), e REMINDER pode ser criado por qualquer usuário organizando seu próprio trabalho sem afetar outros. Resolução de WARNING e ISSUE preenche resolved_at timestamp e resolved_by_account_id removendo anotação de listas de pendências mas preservando no histórico completo, apenas criador original ou usuário com role superior pode marcar como resolvido (field agent que criou WARNING pode resolver, ou MANAGER pode resolver WARNING de qualquer usuário), sistema pode exigir comentário explicando resolução em Annotation filha (reply thread) documentando o que foi feito para corrigir problema. Filtros de visualização permitem mostrar apenas issues não resolvidos, apenas warnings, ocultar notes mantendo interface focada em problemas acionáveis, e busca textual em content de anotações localiza discussões sobre tópicos específicos independente de tipo. Futuramente sistema pode ter tipos adicionais como QUESTION para perguntas que aguardam resposta de outro usuário, APPROVAL_REQUEST para solicitações formais de aprovação que exigem ação de superior, e AUDIT_NOTE para observações de auditoria que são imutáveis e não podem ser deletadas ou editadas garantindo integridade de trilha de auditoria.

**Módulos:** GEOAPI, GEOWEB, REURBCAD, GEOGIS
