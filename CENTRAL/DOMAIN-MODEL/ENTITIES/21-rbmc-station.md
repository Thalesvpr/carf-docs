# RbmcStation (Estação RBMC IBGE)

Entidade representando estação de referência da Rede Brasileira de Monitoramento Contínuo RBMC operada pelo IBGE fornecendo dados de correção diferencial para pós-processamento GPS. Campos conceituais incluem identificador único global, station_code código oficial de 4 letras identificando estação conforme nomenclatura IBGE (RIOD para Rio de Janeiro, SPAR para São Paulo Aeroporto, RECF para Recife), name nome descritivo completo da estação, state_uf sigla do estado onde estação está localizada para busca geográfica, latitude coordenada precisa da antena da estação em graus decimais (precisão geodésica milimétrica), longitude coordenada precisa da antena, altitude elevação elipsoidal precisa em metros, operator entidade responsável pela operação geralmente IBGE ou parceiros (universidades órgãos estaduais), url_rinex_data URL base para download de arquivos de observações RINEX da estação (FTP ou HTTP do IBGE), active booleano indicando se estação está operacional e transmitindo dados atualizados, last_data_available timestamp da última observação disponível para monitoramento de funcionamento, equipment descrição do receptor e antena instalados na estação para referência técnica, e metadados de instalação (installation_date para calcular idade de equipamento). Relacionamentos incluem é utilizada por múltiplos SurveyProcessing jobs de pós-processamento (1:N onde topógrafos escolhem estação mais próxima de seu levantamento), e indiretamente relaciona-se com SurveyPoints através de processamento diferencial aplicado. Regras de negócio estabelecem que station_code deve corresponder a estação real da rede RBMC validado opcionalmente via consulta a lista oficial do IBGE, coordenadas da estação são geodésicas de alta precisão (sub-centímetro) estabelecidas por campanhas de observação longas e ajustamento rigoroso servindo como referência para todos processamentos, distância entre estação RBMC e pontos coletados em campo idealmente não deve exceder 300km para pós-processamento de qualidade (quanto mais próximo melhor correção troposférica e ionosférica), dados RINEX devem estar disponíveis para período de coleta dos SurveyPoints (sistema alerta se dados da estação não estão disponíveis para data de observação impossibilitando processamento), e estação inativa (active=false) não deve ser sugerida em workflows de pós-processamento mas permanece cadastrada para reprocessamento histórico de dados. Workflow típico inicia com cadastro de estações RBMC relevantes para área de atuação do tenant (prefeitura do Rio cadastra estações RIOD e vizinhas, empresa de topografia nacional cadastra rede completa), sistema mantém lista atualizada verificando periodicamente disponibilidade via url_rinex_data detectando estações offline, ao executar pós-processamento diferencial via SurveyProcessing topógrafo ou sistema automaticamente seleciona estação RBMC mais próxima dos pontos coletados em campo (query espacial por distância), sistema baixa arquivo RINEX da estação para período de observação via URL configurada (exemplo: RIOD observações do dia 2025-01-05 das 10h às 11h), software de pós-processamento combina observações da estação de referência com observações do receptor móvel em campo calculando diferenças de fase de portadora e resolvendo ambiguidades inteiras para determinar linha de base precisa (vetor 3D entre estação e ponto coletado), resultado é coordenada corrigida do ponto com acurácia centimétrica e relatório de precisão estimada incluindo DOP (Diluição de Precisão) e RMS (erro quadrático médio). Sistema pode ter cache local de arquivos RINEX mais usados reduzindo downloads repetidos da mesma estação e dia, integração automatizada com FTP do IBGE baixa dados necessários on-demand sem intervenção manual do topógrafo, e validação de qualidade verifica que dados RINEX baixados estão completos e não corrompidos antes de iniciar processamento (header válido, observações contínuas sem gaps grandes). Seleção de estação otimizada considera não apenas distância mas também disponibilidade de dados (preferir estação um pouco mais distante mas com dados completos do que estação próxima com gaps de observação), e futuramente sistema pode suportar processamento multi-estação usando dados de várias RBMC simultaneamente para melhorar robustez e acurácia especialmente em levantamentos de longa baseline.

**Módulos:** GEOAPI, GEOWEB, REURBCAD, GEOGIS
