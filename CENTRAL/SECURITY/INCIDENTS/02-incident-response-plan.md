# Incident Response Plan

Plano de resposta a incidentes de segurança do CARF seguindo framework NIST (Preparation, Detection, Analysis, Containment, Eradication, Recovery, Post-Incident). Preparation mantém equipe resposta treinada (Security Lead, SysAdmin, DBA, DevOps, Legal) com runbooks atualizados, ferramentas prontas (Wireshark packet capture, forensics disk imaging, backup restore scripts), e contacts externos (cyber insurance, FBI/PF se crime, lawyer consultoria legal). Detection via monitoring contínuo (Prometheus alerts anomalias CPU/memory, SIEM correlacionando logs suspicious patterns, IDS/IPS detectando network intrusions, users reportando phishing via botão Report Suspicious) triggering incident workflow automaticamente criando ticket PagerDuty escalando on-call. Analysis investiga escopo (quantos usuários afetados, quais dados comprometidos, vetor ataque utilizado, indicadores comprometimento IOCs) preservando evidências forenses (snapshot VMs, dump memory, export logs imutáveis) chain of custody legal. Containment isola sistemas afetados (disconnect network, block IPs atacantes firewall, revoke tokens comprometidos, disable contas suspeitas) minimizando dano enquanto investiga, short-term containment imediato e long-term containment reconfigurando arquitetura. Eradication remove causa raiz (patch vulnerabilidade explorada, remove malware, close backdoors, harden configs) verificando nenhum persistence mechanism permanece. Recovery restaura operação normal (restore backups validados limpos, redeploy sistemas hardened, gradual rollout monitorando recorrência) verificando funcionalidade completa. Post-incident documenta lessons learned (root cause analysis, timeline ataque, gaps detectados, ações preventivas futuras) publicando post-mortem interno melhorando processos.
