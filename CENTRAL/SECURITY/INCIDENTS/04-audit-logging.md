# Audit Logging

Logging de auditoria do CARF rastreando ações críticas para compliance, troubleshooting, e detecção ameaças. Eventos auditados incluem autenticação (login sucesso/falha, logout, MFA bypass, password reset, session timeout), autorização (acesso negado 403, privilege escalation tentativa, role assignment mudança), data access (criação/leitura/atualização/delete entidades críticas Units/Holders, export relatórios, query dados sensíveis CPF), e administrative actions (config changes, user management, database schema migrations, deploy production). Formato logs estruturado JSON facilitando parsing automático com campos padronizados (timestamp ISO8601 UTC, severity level, event_type, actor user_id/service_account, tenant_id isolando multi-tenancy, action performed, resource_type resource_id afetado, ip_address source, user_agent client, request_id correlacionando distributed tracing, e result success/failure com error_code se falha). Armazenamento logs centralizado via stack ELK (Elasticsearch indexando, Logstash agregando, Kibana visualizando) ou cloud-native (AWS CloudWatch, Azure Monitor) com retention 5 anos compliance LGPD, immutability garantindo logs não alteráveis pós-escrita detectando tampering, e encryption at-rest protegendo dados sensíveis. Alertas automáticos Prometheus detectando padrões suspeitos (spike failed logins mesma conta, acesso horário incomum 3am, export massa dados, privilege escalation não autorizado, geographic anomaly login Brasil seguido China 5min) triggerando investigation security team. SIEM correlaciona eventos cross-systems detectando ataques complexos (brute force distribuído, lateral movement network, data exfiltration gradual) agregando inteligência ameaças feeds externos.
