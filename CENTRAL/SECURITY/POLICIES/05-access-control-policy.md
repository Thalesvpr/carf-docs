# Access Control Policy

Política de controle de acesso do CARF gerenciando ciclo de vida usuários e permissões. Onboarding novos usuários via workflow aprovado (solicitação → aprovação gestor → criação conta Keycloak → atribuição role apropriado → treinamento obrigatório segurança antes acesso produção), provisioning automático via integração LDAP/AD copiando usuários Active Directory corporativo sincronizando roles, e acesso inicial mínimo expandindo conforme necessidade justificada. Revisão periódica acessos trimestral auditando usuários ativos removendo inativos >90 dias, validando permissões ainda necessárias função atual, e recertificação gestores confirmando subordinados precisam acessos atribuídos. Offboarding imediato desligamento colaborador (desativar conta Keycloak em 1h notificação RH, revogar tokens ativos forçando logout, transferir ownership dados para substituto, e arquivar logs auditoria usuário 5 anos compliance). Acesso privilegiado (ADMIN, DBA) protegido adicionalmente com MFA obrigatório, sessões reduzidas 15min inatividade, approval dual para operações críticas (delete database, alterar RLS policies), e logging detalhado comandos executados rastreando responsabilidade. Acesso remoto VPN obrigatória para acessar produção fora rede corporativa, IP whitelisting restringindo acesso APIs administrativas, e bastion hosts intermediando acesso direto banco produção auditando queries executadas. Contas serviço (API keys, service accounts) rotacionadas a cada 90 dias, scoped permissões mínimas necessárias, e monitoradas uso anormal (spike requests, acesso horários incomuns) alertando potencial comprometimento.
