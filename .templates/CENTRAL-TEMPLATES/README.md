# TEMPLATES

Templates padronizados garantindo consistência documentação CARF incluindo RF-template.md estrutura para Requisitos Funcionais com metadados YAML descrição critérios aceitação checklist rastreabilidade UC-template.md estrutura para Use Cases com atores pré pós-condições fluxo principal fluxos alternativos exceção rastreabilidade US-template.md estrutura para User Stories com metadados YAML formato Como Quero Para critérios Gherkin Given When Then rastreabilidade e README-template.md estrutura padrão READMEs pastas com título resumo objetivo navegação última atualização. Todos templates incluem exemplos preenchidos instruções uso seguindo prevention-standards evitando hardcoding duplicação mantendo single source of truth SSOT principle consistency accuracy maintainability.

## Templates Disponíveis

### Requisitos e Documentação
- [RF-template.md](./RF-template.md) - Template para Requisitos Funcionais com metadados e rastreabilidade
- [UC-template.md](./UC-template.md) - Template para Use Cases com fluxos principais e alternativos
- [US-template.md](./US-template.md) - Template para User Stories formato Gherkin
- [README-template.md](./README-template.md) - Template padrão para READMEs de pastas

### Documentação de Projeto
- [PROJECT-DOCS-TEMPLATE.md](./PROJECT-DOCS-TEMPLATE.md) - Template completo para estrutura DOCS de projetos
